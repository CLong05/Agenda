三、实验心得
本次实验耗时较长，但也较有收获:
  首先是需要管理变化的数据---用户和会议；
  其次是多态实现，一共有7个指令，通过接口与命令实例化为对应的命令类，从而
执行命令；
  另外，是将业务逻辑层的处理结果和表示层的显示方式互相隔绝开。对于执行结果的
显示，Comand的exec会返回int编号，表示层根据接收到的编号，选择显示的方式，
例如命令行交互显示，图形界面显示等。在本实验中使用命令行交互显示的方式；
  最后是采用微软推荐的分层式结构进行设计，从下至上分别为:数据访问层、业务
逻辑层、表示层，使得设计更有条理、更加合理。其中，数据访问层本次实验没有
要求，所以没有实现。
